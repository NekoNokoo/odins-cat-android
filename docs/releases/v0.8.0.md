# Odin's Cat 0.8.0

`0.8.0` закрепляет новую bridge-цепочку до рабочего end-to-end состояния: `device -> Yandex VM :443 -> xhttp -> REALITY -> origin -> internet`.

## Что главное в этом релизе

- `Yandex edge` теперь разворачивается как настоящий `xhttp -> REALITY` bridge, а не как старый passthrough.
- Android runtime для `Yandex edge` стабильно поднимается как `cdn-anti-whitelist`, а не уезжает в legacy `reality-vps-lab`.
- `edge attach` больше не требует `socat` в `xray-proxy` пути.
- `cover-host pool` и `ordered fallback` для `Yandex camo` уже встроены в runtime.
- диагностические логи по выбранному front и active chain стали заметно понятнее.

## Что это даёт на практике

- новая `Yandex VM` реально завершает первый hop на `:443`;
- дальше edge сам открывает свежий `REALITY`-hop на origin;
- цепочка подтверждается с трёх сторон:
  - телефон,
  - Yandex VM,
  - origin VPS.
